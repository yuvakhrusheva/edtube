# Проект: YouTube Quize-Mode Extension (MVP)

> Актуально на v0.1.0. Технические детали: [README.md](README.md), [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).

## 1. Описание продукта

Расширение для Google Chrome, которое превращает просмотр обучающих видео на YouTube в интерактивный курс в стиле Coursera. Расширение работает в асинхронном режиме **«Учеба»** (включается строго по кнопке пользователя), автоматически останавливает видео на логических паузах и задаёт «умные» вопросы по пройденному материалу для проверки понимания и удержания фокуса.

## 2. Стек технологий

| Слой | Технологии |
|------|------------|
| **Расширение** | Vanilla JS, Chrome Extension API MV3, HTML5, CSS3 |
| **YouTube** | Content script, page-bridge (Innertube, timedtext, srv3) |
| **Backend API** | Ваш сервер (планируется): REST, session auth |
| **LLM** | **DeepSeek** на **сервере** (OpenAI-compatible API); опционально OpenAI/Gemini, Helicone |
| **Ключи API** | Только в `.env` backend — см. [`.env.example`](.env.example) |

Пользователям **не нужно** вводить ключи LLM. Расширение — клиент вашего API.

## 4. Логика работы (User Flow)

### 4.1 Идентификация и активация

1. На watch-странице расширение получает **Google user ID** через `chrome.identity` или **anonymous UUID** (`chrome.storage.local`) — для метрик и backend `user_id`.
2. Пользователь включает тумблер **«Режим учёбы»** в popup (`studyModeEnabled`).
3. Если тумблер выключен — расширение не загружает субтитры и не вмешивается в YouTube.

### 4.2 Загрузка субтитров (реализовано)

1. При включённом режиме «Учеба» на watch-странице `content.js` запрашивает полный транскрипт с таймкодами.
2. Выбор дорожки: **ручные субтитры → ASR**, язык = язык аудио видео.
3. Формат сегментов: `{ startMs, durationMs, text }`.
4. При SPA-навигации (смена видео в плейлисте) транскрипт перезагружается (`yt-navigate-finish`).
5. Отладка: Console → `[Quize-Mode] Transcript loaded:` (preview первых 3 сегментов).

### 4.3 Асинхронная предзагрузка и умная пауза (планируется)

1. `content.js` отправляет чанк транскрипта (~7 мин видео) в **background.js**.
2. `background.js` вызывает **ваш backend API** (`POST /v1/quiz/generate`) с `userId` и chunk.
3. Backend вызывает LLM, проверяет дневную квоту, возвращает вопрос и `pauseTimestampMs`.
4. Расширение ставит триггер на таймкод; при достижении — пауза видео.

### 4.4 Интерфейс теста (планируется)

1. Оверлей поверх плеера (glassmorphism): вопрос Multiple Choice, 4 варианта.
2. **Правильно:** зелёный маркер, «Продолжить», play; фоновая подготовка следующего вопроса.
3. **Неправильно:** подсветка верного ответа, «Вернуться назад» — перемотка на начало смыслового блока.

### 4.5 Квота и Premium (планируется)

- Free tier: N вопросов/день на `user_id` — **источник правды на backend**.
- Popup: счётчик «осталось X вопросов», кнопка «Купить Premium».
- Premium: повышенный лимит или без лимита — флаг `plan` на сервере.

## 5. Backend API (контракт для интеграции)

Ключи: [`.env.example`](.env.example) → скопировать в `.env`.

### Auth bootstrap

```
POST /v1/auth/bootstrap
{ "userId": "...", "userSource": "google" | "anonymous" }
→ { "sessionToken": "...", "expiresAt": "ISO-8601" }
```

### Генерация вопроса

```
POST /v1/quiz/generate
Authorization: Bearer <sessionToken>

{
  "videoId": "zwSikeU-STI",
  "transcriptChunk": [{ "startMs": 0, "durationMs": 2000, "text": "..." }],
  "language": "ru"
}
```

Ответ — JSON из раздела 6 (промпт LLM) плюс поле `quota`.

## 6. Системный промпт для LLM

Запросы выполняются **на backend** к DeepSeek (`deepseek-chat` по умолчанию, см. `LLM_PROVIDER=deepseek` и `LLM_MODEL` в `.env`). API совместим с OpenAI Chat Completions: `DEEPSEEK_BASE_URL` + `DEEPSEEK_API_KEY`.

```
Ты — опытный методист онлайн-обучения. Проанализируй кусок предложенного транскрипта YouTube-видео и создай ОДИН вопрос с четырьмя вариантами ответа (Multiple Choice) для проверки понимания.

Критерии качества:
1. Игнорируй приветствия, рекламу, воду, шутки и призывы подписаться. Вопрос должен быть строго по сути, фактам и логике обучающего контента.
2. Найди идеальный логический конец мысли в интервале и верни точный таймкод для паузы (pause_timestamp).
3. Если в предоставленном тексте вообще нет полезной обучающей информации (идет сплошная реклама или пустая болтовня), верни JSON со статусом "skip".

Формат ответа — строго валидный JSON (без markdown-разметки):
{
  "status": "success",
  "pause_timestamp": "05:42",
  "question": "Текст вопроса?",
  "options": ["Вариант A", "Вариант B", "Вариант C", "Вариант D"],
  "correct_index": 0
}

При status "skip":
{
  "status": "skip"
}
```

Backend конвертирует `pause_timestamp` в `pauseTimestampMs` для extension.

**Реализация (актуально):** один generate возвращает **3 вопроса** в массиве `questions[]`; чанк = 5 мин. Промпт в [`server/src/prompts/quiz.js`](server/src/prompts/quiz.js).
